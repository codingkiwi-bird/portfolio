petbuddy

반려견 성향 분석 · 매칭 플랫폼

강아지 사진 기반 종 판별후 시터와 매칭을 도와주는 모바일 플랫폼

AWS EC2 AWS ALB AWS Rekognition

프로젝트 개요

petbuddy는 강아지 사진을 업로드하면 AWS Rekognition으로 품종을 추론하고, 동물행동학 관련 논문에서 제시된 지표를 수학적 산식으로 모델링하여 품종 특성과 관찰 데이터에 따른 성격(활동성, 사회성, 민감도 등)을 점수화합니다. 산출된 성격 프로파일을 기반으로 매칭까지 도와주는 서비스입니다.

4인 개발

서버 아키텍처 설계 및 데이터베이스 연결과 논문기반 DBTI 산식 함수화 담당

주요 기능

  • AWS Rekognition을 통한 품종 판별
  • 논문 기반 수학적 산식으로 성격 지표(활동성·사회성·민감도 등) 산출
  • 실시간 채팅 및 알림(Socket.io)
  • 예약/요청 관리, 프로필/이력 관리
  • 모바일 앱 경험 제공(React Native)

AWS 아키텍처

알고리즘 / 산식 개요

목표는 반려견 성향의 DBTI 유형을 산출하는 것입니다. 전체 파이프라인은 다음과 같습니다.

  • 사진 입력 → AWS Rekognition으로 품종 분석
  • 품종 → 16개 성격 지표 중 DBTI 산출에 필요한 11개 지표로 치환 (Kaggle 데이터 활용)
  • 표준화 → 11개 지표 각각을 Z-score로 표준화
  • 가중 합산 → 지표별 가중치(근거: C-BARQ)를 곱해 합산하여 최종 점수 계산
  • 유형 결정 → 각 축의 부호/크기를 기준으로 DBTI 유형을 판정

데모 영상

개발 과정에서 배운 점

AWS 상에서의 실서비스 아키텍팅 전 과정을 설계·구현하며 다음을 체계화했습니다.

  • VPC 분리 설계: 퍼블릭/프라이빗 서브넷, ALB 앞단 배치, NAT 경유 아웃바운드 구성
  • 컴퓨트/네트워킹: EC2 오토스케일 전략, 헬스체크/롤링 배포, 보안그룹·NACL 최소권한
  • 보안/운영: 멀티 AZ, IAM 역할 기반 권한, S3 정적 자산 퍼블리싱, ACM 인증서와 Route53 연동

이를 통해 요구사항을 인프라 설계 원칙으로 변환하는 방법과, 배포·관측·보안을 표준화하여 운영하는 방법을 배웠습니다.
